# 习惯约定

> 本仓库在协作/填充配方过程中形成的习惯,和 [目录设计文档](python-cookbook-目录设计.md) 里的
> "贯穿全库的约定"互补——那边讲**结构规则**,这边讲**动手时的习惯**。

---

## 目录与结构

1. **用到再建,不建空壳** —— 目录做到哪块才建哪块,不预先铺一堆空目录占位
   (空目录 git 本来也不跟踪)。设计文档里列出的是"规划",不代表磁盘上都得先建出来。
2. **所有层级目录带两位序号前缀**(`01_ 02_ ...`),阅读顺序即学习/工程顺序;
   工程约定目录(`tests/`、`.github/`)按惯例不加号。
3. **一个库/主题一个子目录,内部按功能拆成多个带序号 `.py`** ——
   以 [`03_stdlib/01_argparse/`](03_stdlib/01_argparse/) 为范式(basic/types/actions/...),
   **不平铺成单个大文件**。

## 代码配方

4. **纯语法用可运行 `.py` + `assert` 自验** —— 每个文件能直接 `python xx.py` 跑,
   内部断言证明自己正确;报错分支用 `SystemExit` / `redirect_stderr` 捕获,不污染输出。
5. **文件顶部注释写清**:所属标准库/三方库、Python 版本、运行或安装方式。
6. **注释和 docstring 一律用中文**。
7. **给的 demo 必须是可执行的**,写完实际跑一遍确认 assert 通过再算完成。

## 命名

8. **项目名 `pycookbook`**(不是 python-cookbook)。
9. **别把顶层命名为 `python` 或 `test`** —— 会和解释器概念、pytest/标准库 `test` 混淆。
